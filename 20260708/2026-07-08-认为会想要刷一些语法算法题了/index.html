笔者是控制的学生，在大一开学时就在学 C，在 20 天速通完 C 后在洛谷上刷了一下基础的语法题（应付计院实验室面试 qwq，不过进度很慢就是了），在了解到 AIoT 组和我们电院实验室没有大关联后，便放弃了加入计院实验室的想法，于是就在学基础的单片机（51 -> 32 这样，不过造轮子的心态让我的进度很慢呢，主要是花在查数据手册上，当然也确实感觉有点开窍了）。

在学期末实验室的小伙伴想要打智能车，笔者欣然前往，是龙芯组（在 C/CPP 基础 && MCU 基础都不稳固的情况下尝试跑 Linux 的 SoC 真是……），在寒假在家按照龙邱的教程装好了 wsl2->ubuntu24，配好了环境，试着学了写 CMakeLists.txt，学了下基础的 cpp（STL 库，类的基础之类的）（其实好像不需要学写 CMakeLists.txt（？）因为官方的库里已经有了，只需要改一下路径而已，我十分想要贴链接不过我删库了。。）。

在大一下学期就在试着写智能车控制部分的代码了，这里贴上官方库的链接：Loongson_2k300_301_Library。初次构想工程时感觉是吃了屎般的难受，因为不知道该从哪里开始，虽然看了龙邱的 Linux 教程还是很难受，比如说线程那一块，还真的会想自己手搓线程调度器的封装来着，点名表扬 lq_base_class.cpp 以及像 std::function<void()> 这样的，孩子们我完全看不懂
（wc 刚刚查了一下 function<void()>

怎么有回调函数？？？中断不是我们 MCU 裸机里面的概念吗 666）。

还有还有，虽然笔者给自己设定了控制方向的身份（而不是视觉）但是学期初我还完全没有系统整体的概念，只知道电控和 PID 的概念啥啥的，谢谢铁头山羊的平衡车教程（bilibili 视频）以及众多博客作者，让我对控制系统有了基本的概念（传递函数和波特图啥啥的，以及进行参数整定的波特图上的目的），虽然只是空有理论而无实践（在试着自己写由编码器和电机 PWM 输出组成的速度环的类时仍然感觉很难受，因为自己之前从来没有接触过这么大的工程，尽管有 20 届龙芯组学长的代码以及一位学长的通用 PID 文件，附件 pid.h 和 pid.cpp 没错我们的库是 Smart-Car）。

笔者为了专心，以及不那么累，就选择了只参加智能车这一个比赛，而不像实验室的一位精力充沛，四处逢源的同学那样（算是押宝了是吗 2333），当然还有由他组织的校航模队（与我的理想距离最近……）。不过呢，我其实在这个项目中没有帮上什么忙，完全是被学长带飞了 qwq，拿了省三。

我会想要搞一下 CPP 基础，不同于在微信低质公众号上看的语言特性的文章。一是我有时做语法题都会出错！！！二是想要学一下多文件项目的撰写（其实智能车是能画出系统框图和数据流的 2333，这里应该看一下设计模式的应该），三是还是想要见识一下吧，虽然我不走算法竞赛。

大概的规划是，暑假先系统地速通泛函和自控（主要是经典控制理论），然后在牛客网上刷技能树。嗯，大概是这样。
